GeoNetwork - Comenzando

Ejecutar la aplicación

GeoNetwork opensource es un Catálogo de Datos Espaciales. Para utilizarlo debe iniciar previamente el servidor web incluido.

En Windows, abrir a la carpeta GeoNetwork opensource, en el menú de inicio, que se crea durante la instalación y ejecutar "Start server". Entonces puede abrir GeoNetwork en su navegador web.

Por defecto, ha de acceder a http://localhost:8080/geonetwork

IMPORTANTE!! Después de abrir la página de inicio de GeoNetwork, validese como Administrador. Debe de usar el nombre de usuario admin y la clave de acceso admin. En el módulo de administración puede modificar su cuenta de usuario.

La documentación online está disponible desde la página de inicio de GeoNetwork. Seleccione Ayuda en la barra de menú. Un enlace a la versión imprimible en formato PDF se puede encontrar al final de la página de ayuda.

Puede iniciar/parar el servidor web Jetty de GeoNetwork usando los scripts ubicados en el directorio $INSTALL_PATH/bin.

En Windows:

  • start-geonetwork.bat
  • stop-geonetwork.bat
En Linux o OSX:
  • start-geonetwork.sh
  • stop-geonetwork.sh

Nota: El entorno de ejecución de Java (JRE) @javaVersion@ Standard Edition (descargar de http://java.sun.com) se requiere para poder ejecutar GeoNetwork opensource. Vea las secciones de resolución de problemas si la instalación o el inicio de GeoNetwork falla.

Cargando metadatos

El catálogo incluye plantillas de metadatos, de forma que pueda crear nuevos metadatos usando el editor de metadatos de Geonetwork.

Si quiere cargar metadatos, dispone de diversas opciones:

  • Importación de ficheros XML desde el módulo de administración del catalogo
  • Importar de ficheros MEF desde GAST o desde el módulo de administración del catalogo
  • Harvesting

Probablemente el método más conveniente es usar el formato MEF cuyo propósito es el intercambio de metadatos y datos, imagenes de vista previa y privilegios básicos de acceso entre nodos de Geonetwork. Puede encontrar ejemplos de ficheros MEF en el directorio \gast\setup\sample-data. El formato del fichero se describe en el Capítulo 3 del manual de referencia del servidor (en \docs\manuals\). Este formato será soportado por otras aplicaciones geoespaciales.
Para importar y exportar metadatos en Geonetwork, debe iniciar la aplicación GeoNetwork Administrator Survival Tool (GAST).

  • Inicialmente configure su conexión en GeoNetwork configurando los datos del servidor y los detalles de la cuenta de acceso en el menú Options => Config. Nota: Se requiere una cuenta de editor o administrador para que funcione correctamente.
  • Para cargar los metadatos de ejemplo, seleccione la opción Sample data en la sección Database y pulse el botón Import.
  • Para cargar datos en formato MEF, seleccione la opción Import en la sección Management. Seleccione la carpeta que contiene el fichero MEF y pulse el botón Import.

Configuración del sistema

Servidor web

GeoNetwork incluye el servidor web Jetty. Si lo requiere, puede usar adicionalmente Jakarta Tomcat version 5.5 . Debe configurar nuevos contextos en Tomcat que enlacen con las aplicaciones en la carpeta web de Geonetwork.

Base de datos

GeoNetwork incluye un servidor de base de datos interno, McKoi SQL.

Si require utilizar otra base de datos puede usar GAST para configurarla y migrar el contenido de la base de datos McKoi. Asegurese que la base de datos está ejecutandose, con un schema y un usuario configurados. Seleccione la base de datos usando la herramienta GAST. Oracle, MySQL y PostgreSQL están actualmente soportadas. GeoNetwork usa JDBC para acceder a la base de datos, por lo que otras bases de datos deben de funcionar también.

Puede configurar también la base de datos manualmente, aunque es más complejo (Acceda al sitio web de la comunidad GeoNetwork para obtener más detalles)
