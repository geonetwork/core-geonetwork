Getting Started

Running the application

@displayName@ is a web based Spatial Data Catalog written in Java @javaVersion@. Requires having Java @javaVersion@ installed, which can be downloaded from @jreName@ .

To use it, you first have to start the included servlet container (Jetty). However, you can also download the @appName@ WAR file and run that on your own container.

You can start the @appName@ webserver using the shell scripts located in the bin directory. These require administrative privileges and have been named:

  • startup.sh (use startup.bat on Windows)
  • shutdown.sh (use shutdown.bat on Windows)

By default, the @appName@ web interface should be available at http://localhost:8080/geonetwork. If you wish to run the application on a different port, please edit the startup shell script. For example, if you are using the provided Jetty server and you wish to run @appName@ on port 1234, you can add the following Java configuration option: -Djetty.port=1234

IMPORTANT

After opening the @appName@ web interface, log on as Administrator. By default, you should use the username admin and password admin. Please modify the password for this account in the Administration section.

Notes

  • When @appName@ runs for the first time, it could take a little longer to start. By default, @appName@ will use a local database (H2), which takes a while to set up if it doesn't exist yet.
  • By default, H2 will create 3 gn.*.db files in the Jetty folder. You can change the location of the database by setting the -Djdbc.database="path\to\gn" Java option (do not include the .db extension) in the startup shell script. Alternatively, you can also set up @appName@ to work with a remote database (e.g. Oracle).
  • If you plan to remove @appName@ but wish to keep the database, please backup the gn.*.db files. When you (re-)install @appName@ again and put the database files back at the original location, @appName@ should pick it up.

Enjoy!
